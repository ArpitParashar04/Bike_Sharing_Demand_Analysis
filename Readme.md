## Background

Bike sharing systems are a new generation of traditional bike rentals where the whole process
— membership, rental, and return — has become automatic. Through these systems, a user can
easily rent a bike from one location and return it at another. There are currently over 500
bike-sharing programs worldwide, comprising more than 500,000 bicycles. Interest in these systems
has grown significantly due to their important role in traffic, environmental, and public health
issues.

Beyond their real-world utility, the data generated by bike sharing systems is attractive for
research. Unlike other transport services such as buses or subways, the duration of travel and
the exact departure/arrival positions are explicitly recorded. This effectively turns a bike
sharing system into a **virtual sensor network** capable of sensing mobility patterns across a
city — meaning many significant urban events can potentially be detected simply by monitoring
this data.

---

## Dataset

Bike-sharing rental demand is highly correlated with environmental and seasonal conditions —
weather, precipitation, day of week, season, and time of day all affect rental behavior.

The core dataset is the two-year historical log (2011–2012) from the **Capital Bikeshare**
system in Washington, D.C., USA, publicly available at
[capitalbikeshare.com/system-data](http://capitalbikeshare.com/system-data). The raw logs were
aggregated on both an hourly and a daily basis, then merged with corresponding weather and
seasonal information sourced from [freemeteo.com](http://www.freemeteo.com).

## Dataset Characteristics

Both `hour.csv` and `day.csv` share the same fields, **except `hr`**, which only appears in
`hour.csv`.

| Field         | Description                                                                                      |
| --------------- | ---------------------------------------------------------------------------------------------------- |
| `instant`       | Record index                                                                                          |
| `dteday`        | Date                                                                                                   |
| `season`        | Season (1: spring, 2: summer, 3: fall, 4: winter)                                                     |
| `yr`            | Year (0: 2011, 1: 2012)                                                                                |
| `mnth`          | Month (1 to 12)                                                                                        |
| `hr`            | Hour (0 to 23) — *hourly data only*                                                                    |
| `holiday`       | Whether the day is a holiday (source: [dchr.dc.gov](http://dchr.dc.gov/page/holiday-schedule))          |
| `weekday`       | Day of the week                                                                                        |
| `workingday`    | 1 if the day is neither a weekend nor a holiday; 0 otherwise                                           |
| `weathersit`    | Weather situation (see scale below)                                                                    |
| `temp`          | Normalized temperature in Celsius, divided by 41 (max)                                                |
| `atemp`         | Normalized "feels like" temperature in Celsius, divided by 50 (max)                                    |
| `hum`           | Normalized humidity, divided by 100 (max)                                                              |
| `windspeed`     | Normalized wind speed, divided by 67 (max)                                                             |
| `casual`        | Count of casual (non-registered) users                                                                |
| `registered`    | Count of registered users                                                                              |
| `cnt`           | Total rental bike count — sum of `casual` and `registered`                                             |

### `weathersit` Scale

| Value | Description                                                                    |
| ------- | ----------------------------------------------------------------------------------- |
| 1       | Clear, few clouds, partly cloudy                                                     |
| 2       | Mist + cloudy, mist + broken clouds, mist + few clouds, mist                          |
| 3       | Light snow, light rain + thunderstorm + scattered clouds, light rain + scattered clouds |
| 4       | Heavy rain + ice pellets + thunderstorm + mist, snow + fog                             |

## Associated Tasks

- **Regression** — Predict hourly or daily bike rental count (`cnt`) based on environmental and
  seasonal settings.
- **Event & Anomaly Detection** — Rental counts correlate with real-world events in the city,
  which are often traceable via search engines (e.g., a query like *"2012-10-30 washington d.c."*
  returns results related to Hurricane Sandy). Some notable events are identified in [1]. The
  dataset can therefore also serve as a benchmark for anomaly or event-detection algorithms.
